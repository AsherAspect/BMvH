Je ontvangt een lijst met categories, vergezeld met criteria die deze categories toelichten.
De classificaties worden verstrekt aan mensen van het bedrijf die snel willen analyseren wat er in de tekst wordt besproken.

Genereer gestructureerde data in EXACT JSON-formaat met de volgende regels:
1. Het hoogste niveau moet altijd een JSON-object zijn met categories als sleutels (bijv. "22", "30").
2. Elke category moet criteria (bijv. "22.12") als geneste objecten bevatten.
3. Elk criteria-object moet het volgende bevatten:
    - "reasoning": een string
    - "met": een integer (0 of 1)
4. De uitvoer MOET beginnen met `{` en eindigen met `}` en geldige JSON zijn.
5. Zorg ervoor dat de uitvoer geldige JSON is, ZONDER trailing commas na het laatste element in een object of array.
6. Laat GEEN criteria of category weg.
7. De EXACTE JSON-format die moet worden gebruikt:
{
    "category": {
        "criteria": {
            "reasoning": "string",
            "met": int
        }
    }
}
8. Voorbeeld van gestructureerde uitvoer in JSON-format:
{
    "22": {
        "22.12": {
            "reasoning": "Stalen scheepsplaten voor de scheepsromp worden besproken in de meegeleverde tekst.",
            "met": 1
        },
        "22.14": {
            "reasoning": "Er is niet besproken dat de stalen scheepsplaten hoge treksterkte hebben."

        }
    },

            "reasoning": "In de context werd gesproken over het huren van een vrachtwagen voor het vervoeren van cascomateriaal."
            "met": 1
        }
    }
}

- Zorg ervoor dat je uitvoerwaarden voor categorie en criteria overeenkomen met de criteria, anders kan je uitvoer niet verder worden verwerkt omdat de waarden niet langer aan bestaande waarden zijn gekoppeld.

- Combineer NOOIT de waarden van "category" en "criteria" in één naam. Gebruik altijd de originele namen van category en criteria respectievelijk.
